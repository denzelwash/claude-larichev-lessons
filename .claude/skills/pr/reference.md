# Справочник скилла pr

## Заголовок PR

Формат по Conventional Commits: `<type>(<scope>): <описание>` — не более 70 символов.

Допустимые типы: `feat`, `fix`, `refactor`, `perf`, `test`, `docs`, `chore`, `ci`, `build`, `revert`.

Примеры:
- `feat(auth): добавить refresh token`
- `fix(transactions): исправить округление суммы`
- `chore(deps): обновить vite до 5.4`

Если `title` передан как аргумент скилла — использовать его без изменений.

## Тело PR

Секции из `template.md`:

| Секция | Когда включать |
|--------|----------------|
| `## Что сделано` | всегда |
| `## API` | только если изменялся бэкенд (`apps/backend`) |
| `## Чек-лист для ревью` | только для `feat`, `fix`, `refactor` с кодом |

Для типов `chore`, `docs`, `ci`, `build` — секцию «Чек-лист для ревью» **не включать**.

## Пре-чек перед PR

Выполняется через `scripts/validate.sh <base-branch>`. Скрипт проверяет:
1. Ветка запушена в remote и не опережает (`up to date` или `ahead of`).
2. Ветка не отстаёт от `origin/<base>` — иначе нужен rebase.
3. В diff нет случайных файлов: `.env`, `dist/`, `build/`, `*.sqlite`, `*.dump`.

Дополнительно вручную:
- Если в `.claude/plans/` есть план для этой задачи — открыть, отметить выполненные `- [x]`, закоммитить обновление.

## Полный алгоритм

1. Изучить блок **Контекст ветки** из `SKILL.md` — там уже есть текущая ветка, статус и diff.
2. Определить параметры: первый аргумент — `title`, второй — `base-branch` (по умолчанию `main`).
3. Убедиться, что ветка запушена: если нет — `git push -u origin <branch>`.
4. Запустить `bash .claude/skills/pr/scripts/validate.sh <base-branch>`. При ошибке — остановиться и сообщить пользователю.
5. Если есть план в `.claude/plans/` — отметить выполненные пункты `- [x]` и закоммитить.
6. Определить заголовок (из аргумента или сгенерировать по Conventional Commits).
7. Собрать тело PR по `template.md`, опуская ненужные секции (см. таблицу выше).
8. Показать пользователю заголовок и тело, дождаться подтверждения или правок.
9. Создать PR:
   ```
   gh pr create --base <base-branch> --title "..." --body "$(cat <<'EOF'
   ...
   EOF
   )"
   ```
10. Вернуть URL созданного PR.
