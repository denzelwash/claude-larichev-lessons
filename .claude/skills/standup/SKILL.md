## Контекст

- сегодняшняя дата: !`date +%Y-%m-%d`
- день недели сегодня (1=пн … 7=вс): !`date +%u`
- вчерашняя дата: !`date -d "yesterday" +%Y-%m-%d`
- пятница текущей недели (для понедельника): !`date -d "last friday" +%Y-%m-%d`
- текущий пользователь git: !`git config user.name`
- email пользователя git: !`git config user.email`
- текущая ветка: !`git branch --show-current`
- коммиты за вчера (oneline): !`git log --all --since="yesterday 00:00" --until="yesterday 23:59" --pretty=format:"%h %as %s (%an)"`
- коммиты за вчера (stat): !`git log --all --since="yesterday 00:00" --until="yesterday 23:59" --stat --format="commit %h%n%an <%ae>%n%ad%n%n    %s%n"`
- смёрженные PR за вчера: !`git log --all --merges --since="yesterday 00:00" --until="yesterday 23:59" --pretty=format:"%h %s"`
- коммиты за пятницу (oneline, для понедельника): !`git log --all --since="last friday 00:00" --until="last friday 23:59" --pretty=format:"%h %as %s (%an)"`
- коммиты за пятницу (stat, для понедельника): !`git log --all --since="last friday 00:00" --until="last friday 23:59" --stat --format="commit %h%n%an <%ae>%n%ad%n%n    %s%n"`

## Алгоритм

1. **Определить период отчёта.**
   - Если сегодня понедельник (`день недели = 1`) — использовать данные за пятницу и выходные (суббота–воскресенье). Заголовок: «Стендап за пятницу–воскресенье».
   - Иначе — использовать данные за вчера.

2. **Отфильтровать коммиты автора.** Из `oneline`-списка оставить только строки, где автор совпадает с `текущий пользователь git`. Чужие коммиты не включать.

3. **Если коммитов нет** — сообщить об этом явно и предложить запустить скилл с расширенным диапазоном вручную (`git log --since="3 days ago"`).

4. **Разобрать коммиты по типам** (из prefix Conventional Commits):
   - ✨ `feat` — новый функционал
   - 🐛 `fix` — исправление бага
   - ♻️ `refactor` — рефакторинг
   - 📝 `docs` / `chore` — документация, обслуживание
   - 🔀 merge-коммиты — вынести отдельно (какой PR влит)

5. **Сгруппировать по scope** (backend / frontend / shared / auth / prisma / etc.) или по смысловой фиче.

6. **Для каждой группы** описать:
   - **Что сделано** — человеческим языком, не дословно из message.
   - **Где** — модули/файлы из `--stat` (упомянуть ключевые, не перечислять всё).
   - **Зачем** — если понятно из контекста.

7. **Итоговая статистика** — в конце одной строкой: `X коммитов · Y файлов изменено`.

8. **Сформировать отчёт** в формате:

   ```
   ## Стендап за <дата или диапазон> — <имя пользователя>

   ### ✨ Новый функционал
   - ...

   ### 🐛 Баг-фиксы
   - ...

   ### ♻️ Рефакторинг
   - ...

   ### 📝 Документация / chore
   - ...

   ### 🔀 Влитые PR
   - ...

   ---
   _X коммитов · Y файлов изменено_
   ```

   Секции без коммитов — не выводить.

9. Вывести отчёт. Не коммитить и не сохранять в файл, если явно не попросят.

## Заметки

- Если git недоступен или репозиторий не инициализирован — сообщить об ошибке.
- Если у пользователя несколько email/identity — включать коммиты по обеим.
- Merge-коммиты GitHub (`Merge pull request #N`) — выносить в секцию «Влитые PR», не смешивать с обычными.
- Описания писать на русском языке.
